# Dioxus Feature Matrix

This snapshot was made from the current `packages/ui`, `packages/web`, and `packages/desktop` code. Official docs links point at the Dioxus 0.7 documentation.

| # | Feature | Implemented? | Evidence? | Docs |
| --- | --- | --- | --- | --- |
| 1 | Components | ✅ | Shared UI uses `#[component] fn Name(...) -> Element` in files such as [`DeveloperTools`](../packages/ui/src/client/components/developer_tools.rs) and routed pages such as [`Home`](../packages/ui/src/client/pages/home.rs). | [Components](https://dioxuslabs.com/learn/0.7/essentials/ui/components/) |
| 2 | RSX markup | ✅ | Pages and components render HTML-like UI through `rsx!`; see [`Main`](../packages/ui/src/client/components/main.rs) and [`TokenListItem`](../packages/ui/src/client/components/token_list_item.rs). | [RSX](https://dioxuslabs.com/learn/0.7/essentials/ui/rsx/) |
| 3 | Props defaults and children | ✅ | [`Page`](../packages/ui/src/client/components/page.rs) accepts `children: Element` and `#[props(default = false)] will_preload`. | [Component props](https://dioxuslabs.com/learn/0.7/essentials/ui/components/) |
| 4 | Event handlers | ✅ | [`DeveloperTools`](../packages/ui/src/client/components/developer_tools.rs) uses `onclick` for the `DB`, `Server`, theme, and language controls. | [Event handlers](https://dioxuslabs.com/learn/0.7/essentials/basics/event_handlers/) |
| 5 | Asset macro | ✅ | [`Main`](../packages/ui/src/client/components/main.rs), [`About`](../packages/ui/src/client/pages/about.rs), and [`AppLanguage`](../packages/ui/src/client/services/localization_service.rs) use `asset!("/assets/...")` for CSS, images, token icons, and flags. | [Assets](https://dioxuslabs.com/learn/0.7/essentials/ui/assets/) |
| 6 | Document head injection | ✅ | [`Main`](../packages/ui/src/client/components/main.rs), [`TopNavigation`](../packages/ui/src/client/components/top_navigation.rs), and app entrypoints inject stylesheets or favicon through `document::Link`. | [Assets](https://dioxuslabs.com/learn/0.7/essentials/ui/assets/) |
| 7 | Local signals | ✅ | [`DeveloperTools`](../packages/ui/src/client/components/developer_tools.rs), [`Main`](../packages/ui/src/client/components/main.rs), and [`AppLayout`](../packages/ui/src/client/mod.rs) use `use_signal`. | [Signals](https://dioxuslabs.com/learn/0.7/essentials/basics/signals/) |
| 8 | Context state | ✅ | [`AppLayout`](../packages/ui/src/client/mod.rs) provides theme, language, token cache, load request, and toast signals with `use_context_provider`. | [Global context](https://dioxuslabs.com/learn/0.7/essentials/basics/context/) |
| 9 | Effects | ✅ | [`Main`](../packages/ui/src/client/components/main.rs) uses `use_effect` for cache updates, loading toasts, and toast dismissal. | [Effects and memos](https://dioxuslabs.com/learn/0.7/essentials/basics/effects/) |
| 10 | Background tasks | ✅ | [`Main`](../packages/ui/src/client/components/main.rs) and [`DeveloperTools`](../packages/ui/src/client/components/developer_tools.rs) use `spawn(async move { ... })` for timers and async button work. | [Async](https://dioxuslabs.com/learn/0.7/essentials/basics/async/) |
| 11 | Async resources | ✅ | [`Main`](../packages/ui/src/client/components/main.rs) loads tokens with `use_resource` and reruns from token-load request changes. | [Data fetching](https://dioxuslabs.com/learn/0.7/essentials/basics/resources/) |
| 12 | Browser local storage | ✅ | [`storage_service`](../packages/ui/src/client/services/storage_service.rs) uses `web_sys::window().local_storage()` for browser theme, language, and token snapshots. | [Web platform](https://dioxuslabs.com/learn/0.7/guides/platforms/web/) |
| 13 | Typed router | ✅ | [`Route`](../packages/ui/src/client/mod.rs) derives `Routable`, and [`App`](../packages/ui/src/client/app.rs) renders `Router::<Route> {}`. | [Router](https://dioxuslabs.com/learn/0.7/essentials/router/) |
| 14 | Router links | ✅ | [`TopNavigation`](../packages/ui/src/client/components/top_navigation.rs) uses `Link { to: Route::... }` for app navigation. | [Navigation](https://dioxuslabs.com/learn/0.7/essentials/router/navigation/) |
| 15 | Active route reads | ✅ | [`Page`](../packages/ui/src/client/components/page.rs) uses `use_route::<Route>()` to decide which preloaded page is active. | [Routes](https://dioxuslabs.com/learn/0.7/essentials/router/routes/) |
| 16 | Layout route | ✅ | [`Route`](../packages/ui/src/client/mod.rs) uses `#[layout(AppLayout)]` for shared chrome. | [Layouts](https://dioxuslabs.com/learn/0.7/essentials/router/layouts/) |
| 17 | Cross-platform app shell | ✅ | [`web/main.rs`](../packages/web/src/main.rs) and [`desktop/main.rs`](../packages/desktop/src/main.rs) both launch the shared [`ui::App`](../packages/ui/src/client/app.rs). | [Introduction](https://dioxuslabs.com/learn/0.7/) |
| 18 | Web target | ✅ | [`packages/web/Cargo.toml`](../packages/web/Cargo.toml) enables `dioxus/web`. | [Web platform](https://dioxuslabs.com/learn/0.7/guides/platforms/web/) |
| 19 | Desktop target | ✅ | [`packages/desktop/Cargo.toml`](../packages/desktop/Cargo.toml) enables `dioxus/desktop`. | [Desktop platform](https://dioxuslabs.com/learn/0.7/guides/platforms/desktop/) |
| 20 | Internationalization | ✅ | [`localization_service`](../packages/ui/src/client/services/localization_service.rs) initializes Fluent bundles and [`DeveloperTools`](../packages/ui/src/client/components/developer_tools.rs) switches languages. | [Internationalization](https://dioxuslabs.com/learn/0.7/guides/utilities/internationalization/) |
| 22 | Error boundaries | ✅ | [`AppLayout`](../packages/ui/src/client/mod.rs) wraps the shared shell in `ErrorBoundary`, rendering [`AppErrorFallback`](../packages/ui/src/client/components/app_error.rs). | [Error handling](https://dioxuslabs.com/learn/0.7/essentials/basics/error_handling/) |
| 23 | Optional server function call | ✅ | [`try_server_service`](../packages/ui/src/server/services/try_server_service.rs) exposes a small `#[get]` endpoint, and [`DeveloperTools`](../packages/ui/src/client/components/developer_tools.rs) calls it from the `Server` button. | [Server functions](https://dioxuslabs.com/learn/0.7/essentials/fullstack/server_functions/) |
| 24 | Client/native database cache | ✅ | [`token_service`](../packages/ui/src/client/services/token_service.rs) calls [`database_service`](../packages/ui/src/client/services/database_service.rs); browser builds use localStorage snapshots, while non-wasm builds use native SQLite. | [Async](https://dioxuslabs.com/learn/0.7/essentials/basics/async/) |
| x1 | Memoized derived UI | x | Try adding search, sort, or filtered token views with `use_memo`; [`TokenList`](../packages/ui/src/client/components/token_list.rs) is the natural starting point. | [Effects and memos](https://dioxuslabs.com/learn/0.7/essentials/basics/effects/) |
| x2 | Dynamic token detail route | x | Try adding `/token/:id` to [`Route`](../packages/ui/src/client/mod.rs) and linking from [`TokenListItem`](../packages/ui/src/client/components/token_list_item.rs). | [Dynamic routes](https://dioxuslabs.com/learn/0.7/essentials/router/routes/) |
| x3 | Server-backed mutation | x | Try moving a small write path behind a server function near [`try_server_service`](../packages/ui/src/server/services/try_server_service.rs) and calling it from [`DeveloperTools`](../packages/ui/src/client/components/developer_tools.rs). | [Server functions](https://dioxuslabs.com/learn/0.7/essentials/fullstack/server_functions/) |
| x4 | Accessible keyboard shortcuts | x | Try adding focused keyboard interactions to [`TopNavigation`](../packages/ui/src/client/components/top_navigation.rs) or [`DeveloperTools`](../packages/ui/src/client/components/developer_tools.rs). | [Event handlers](https://dioxuslabs.com/learn/0.7/essentials/basics/event_handlers/) |
| x5 | Route-level loading or pending states | x | Try adding route-specific loading affordances around [`Page`](../packages/ui/src/client/components/page.rs) and [`Main`](../packages/ui/src/client/components/main.rs). | [Data fetching](https://dioxuslabs.com/learn/0.7/essentials/basics/resources/) |
| 21 | Forms | ❌ | [`Contact`](../packages/ui/src/client/pages/contact.rs) links to an external form; no in-app Dioxus form submission flow is implemented. | [Forms and multipart](https://dioxuslabs.com/learn/0.7/essentials/fullstack/forms/) |
| 25 | SSR and hydration | ❌ | [`packages/web`](../packages/web/Cargo.toml) has optional `server` feature wiring, but [`App`](../packages/ui/src/client/app.rs) is launched as a client app with no SSR/hydration entrypoint today. | [Server side rendering](https://dioxuslabs.com/learn/0.7/essentials/fullstack/ssr/) |
